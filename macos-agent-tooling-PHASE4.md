## 3. Minor 改进项（来自 Phase 1~3 QA 观察）

| 改进项 | 现状 | 改进后 |
|--------|------|--------|
| **http_request IPv6 覆盖** | 可能仅支持 IPv4 | 增加 `family=socket.AF_UNSPEC` 兼容双栈 |
| **web_search 2000 字符限制** | 响应可能超长 | 截断至 2000 chars，避免 context 溢出 |
| **thinking 多段落** | thinking 输出可能格式混乱 | 支持 `---` 分段展示，增强可读性 |

## 4. Phase 5 方向建议

### Option A: Tool Registry + 安全沙箱（推荐）
完善 Tool Registry，加入：
- 权限分级（read-only / write / dangerous）
- 执行超时控制
- 审计日志

### Option B: Memory 增强
- Hybrid retrieval（BM25 + vector）
- 记忆自动摘要（compression）
- Import/export 记忆

### Option C: SwiftUI 产品化
- Launcher Panel（HUD 入口）
- Menu bar 常驻
- 系统快捷键集成

### 推荐路径
**Phase 5 = Option A（Tool Registry）+ Option C 轻量集成（Menu bar 入口）**
理由：工具权限是 Agent OS 的核心信任边界，应在内测前稳定。
